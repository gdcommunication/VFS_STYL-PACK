Integrationsinformationsfelder

Die Integrationsinformationsfelder enthalten Informationen über die ausgewählte Integration. Je nach Integration können Sie die Informationen bei Bedarf anzeigen oder bearbeiten. Manche Informationen können evtl. nicht bearbeitet werden.

Die folgenden Informationen werden in der Detailansicht für Integration angezeigt.

Feld Beschreibung/Aktion
Integration Der Name der Integration.

Authentifizierungstyp

 

Die Methode zum Authentifizieren von Daten.

Wenn der Authentifizierungstyp geändert wird, schlägt die Integration möglicherweise fehl.

Der externe Service muss von einem einzelnen Service-Benutzer oder mehreren Service-Benutzern autorisiert werden, wenn die Methode OAuth 1 oder OAuth2 ist, die Integration aktiviert ist und Client-ID und Client-Geheimnis festgelegt sind.
Synchronisierungsmethode Legt fest, ob oder wie Informationen zwischen dem integrierten System und Saleslogix synchronisiert werden. Die möglichen Optionen lauten: "Keine Synchronisierung", "Client-seitige Synchronisierung" und "Server-Synchronisierung". Die Server-Synchronisierung gibt an, dass der Saleslogix Integrationssynchronisierungsservice verwendet wird.
Verknüpfungstyp Der Typ der Verknüpfung, der für den Datenaustausch zwischen dem integrierten System und Saleslogix verwendet wird. Wenn "Admin" oder "System" ausgewählt wird, werden die Daten über eine einzelne Verknüpfung verwaltet. Wenn "Benutzer" ausgewählt wird, erfolgt die Datenübertragung für jeden Benutzer über eine separate Verknüpfung.
Client-ID

Nur die Authentifizierungstypen OAuth1 oder OAuth2.

Wird vom Integrationsprovider während der Registrierung zugeordnet.
Geheimnis

Nur die Authentifizierungstypen OAuth1 oder OAuth2.

Wird vom Integrationsprovider während der Registrierung zugeordnet.
Aktiviert

Verwenden Sie diese Option, um die Synchronisierung für diese Integration ein- oder auszuschalten

Wenn diese Option nicht ausgewählt wird, werden keine Daten zwischen dem integrierten System und Saleslogix ausgetauscht. Außerdem sind dann alle folgenden Synchronisierungsoptionen nicht verfügbar.
Sync- Handler

Diese Informationen sind schreibgeschützt.

Definiert einen Nicht-SData-Synchronisierungs-Handler für eine Integration, die ihre eigenen SData-Synchronisierungsendpunkte nicht verfügbar macht.

Der Handler ist verantwortlich für die Zuordnung von Daten zwischen dem Vertrag und der Integration.
Synchronisieren alle Definiert die Anzahl der Minuten, die zwischen Synchronisierungszyklen vergehen sollten.
Synchronisierung ausführen zwischen Definiert die Stunden, zwischen denen die Integration synchronisiert werden soll.
