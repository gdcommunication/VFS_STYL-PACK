Was sind Kontaktprozesse?

Ein Prozess besteht aus einer Reihe von Aufgaben, die in einer bestimmten Reihenfolge innerhalb eines bestimmten Zeitraums ausgeführt werden. Mit Kontaktprozessen können Sie sich wiederholende Aufgaben automatisieren und eine automatisierte und damit einheitliche Kundenbetreuung gewährleisten. Kontaktprozesse werden mit Architect erstellt.

Prozesse werden an Kontakte angehängt. Jede Aufgabe in einem Prozess beginnt nach Abschluss der vorherigen Aufgabe, es sei denn, es wurde ein Zeitabstand festgelegt. Durch das Verknüpfen eines Prozesses mit einem Kontakt wird der Prozess nicht automatisch gestartet. Prozesse müssen in den Saleslogix Network Client- oder Saleslogix Remote Client-Anwendungen gestartet werden.

Mithilfe des Saleslogix Web Kontaktprozess-Managers können Sie den Status der Ihren Kontakten zugewiesenen Prozesse anzeigen. Alle weiteren Änderungen an Prozessen müssen im Saleslogix Network Client oder Saleslogix Remote Client vorgenommen werden. Verwenden Sie die Registerkarte Prozesse in der Kontaktdetailansicht, um die mit einem einzelnen Kontakt verknüpften Prozesse anzuzeigen.

Durch Ihren Sicherheitszugriff wird bestimmt, welche Funktionen verfügbar sind. Wenden Sie sich an den Administrator, wenn Sie Ihre Zugriffsrechte ändern möchten. Für WebViewer-Benutzer kann die Funktionalität eingeschränkt sein.
