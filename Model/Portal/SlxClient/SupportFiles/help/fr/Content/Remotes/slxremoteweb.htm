Présentation de Saleslogix Client Web hors ligne

Utilisateurs distants uniquement

Le Client Web hors ligne fonctionne comme le Client distant Saleslogix si ce n'est que les utilisateurs se connectent à leur base de données locale via le client Web. Le Client Web hors ligne vous permet de travailler hors ligne sans connexion Internet active.

Les modifications entre la base données de l'utilisateur du et la base de données du bureau principal sont transférées grâce à la synchronisationClient Web hors ligne. Pour synchroniser les modifications entre votre base de données et la base de données du bureau principal, vous devez utiliser le client de synchronisation. Pour plus d'informations, reportez-vous à l'Aide en ligne du client de synchronisation.

Pour exécuter des rapports dans un état déconnecté, le Client Web hors ligne doit être exécuté sur le serveur Windows 2003 avec IIS. Le Saleslogix Webserveur Reporting doivent également être installés et configurés. Si ces conditions ne sont pas remplies, les utilisateurs du client Web déconnectés ne peuvent exécuter Web Reporting que s'ils sont connectés au serveur Web Reporting hôte. Ces utilisateurs verront des rapports qui représentent les données de la base de données hôte et non de leur base de données distante.

Quelle est la différence dans le Client Web hors ligne ?

  • La vue des Nouveautés permet aux utilisateurs du Client Web hors ligne de s'abonner et de se désabonner des comptes.
  • (Intégration de la comptabilité uniquement) La synchronisation du contrat entre Saleslogix et un système de comptabilité pris en charge a lieu dans la base de données hôte. Par conséquent, toutes les modifications de Client Web hors ligne ou du système de comptabilité doivent être synchronisées via la base de données hôte.
