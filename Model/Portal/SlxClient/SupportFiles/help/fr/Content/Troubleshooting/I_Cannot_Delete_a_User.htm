Impossible de supprimer un utilisateur

Message d'erreur : D'autres utilisateurs sont subordonnés à cet utilisateur. Vous ne pouvez pas le supprimer.

Solution : Désélectionnez la case "L'Utilisateur est un responsable" dans la vue Détails de l'utilisateur à supprimer.

 

Message d'erreur : Cet utilisateur possède actuellement des enregistrements qui doivent être réaffectés avant de le supprimer, ou l'utilisateur peut être propriétaire d'une équipe : dans ce cas, il faut le retirer de l'équipe pour pouvoir le supprimer.

Solution : Exécutez le réalignement du territoire.

  • Si ce message s'affiche après l'exécution d'un réalignement de territoire, vous devrez peut-être exécuter le contrôleur d'intégrité pour remplacer la valeur du champ IDCODESEC dans différentes tables par la valeur correcte. Ceci peut être réalisé en utilisant le Contrôleur d'intégrité, les tests suivants étant activés : « Synthèse de compte avec propriétaire erroné (défini sur Propriétaire du compte) », « Contacts avec le mauvais propriétaire (défini sur Propriétaire du compte) » et « Opportunités dont le propriétaire est incorrect (défini sur Propriétaire du compte) ».
  • Le message précédent peut également s'afficher lorsque l'utilisateur est le propriétaire d'une équipe. Dans ce cas, vous devez désigner un autre utilisateur comme propriétaire de l'équipe.
  • Une autre cause expliquant ce message est que le réalignement de territoire a échoué lors d'une précédente exécution et que cet utilisateur possède toujours des enregistrements dans la table CONTACT, ACCOUNT, ACCOUNTSUMMARY ou OPPORTUNITY. La syntaxe SQL suivante affichant les utilisateurs et leurs valeurs IDCODESEC respectives peut s'avérer utile :

SELECT UI.USERID, UI.USERNAME, SR.SECCODEID, SC.SECCODEDESC

FROM SECRIGHTS SR, USERINFO UI, SECCODE SC

WHERE UI.USERID = SR.ACCESSID AND

SR.SECCODEID = SC.SECCODEID

ORDER BY ACCESSID
